# Security Lab

Public repository for my personal security lab and hands-on cybersecurity project documentation.

The main purpose of this repository is to document practical security-focused implementations, validation steps, and lessons learned in a structured and reviewable way.

## Documentation

The full project documentation is published here:

**[Open the documentation site](https://fe0ix.github.io/security-lab/)**

## Current Scope

This repository currently focuses on:

- SIEM deployment and alert validation
- Endpoint visibility and telemetry improvements
- Active Directory-related monitoring use cases
- Network segmentation and firewall policy design
- Security-focused lab documentation and evidence collection

## Current Project

- **Wazuh Lab** — deployment and documentation of a dedicated monitoring environment for log collection, alerting, and security validation

## Purpose

This lab is used to build practical experience in areas relevant to Security Analyst and Blue Team work, with an emphasis on implementation, testing, troubleshooting, and documentation rather than theory alone.

## Notes

This is a public portfolio repository. Sensitive information such as credentials, API keys, personal data, and environment-specific identifiers is removed, generalized, or sanitized before publication.

AI tools were used in parts of the documentation workflow for drafting, brainstorming, and wording support. All published content was reviewed, adapted, and validated by me before release.
